# Tappo App Intelligent Visual Anchor Decision - 2026-06-08

## Decision

Use a hybrid intelligent visual anchor.

Important clarification from the user: `小T助手也并不是固定的`.

This means Tappo Phone should not lock the whole App to one fixed mascot. The intelligent visual language is a flexible `assistant expression layer`, not a permanent character that must appear everywhere.

## Anchor System

### 1. Flexible 小T Assistant

Use when the page needs warmth, guidance, onboarding, or human-like assistance.

Allowed pages:

- App cover / splash.
- Role selection.
- Owner home opening area.
- Staff home / app center.
- Empty states.
- Help and guidance moments.

Rules:

- 小T can vary by expression, pose, crop, or even abstract avatar form.
- 小T should not replace the Tappo logo.
- 小T should not become a repeated large decoration on every page.
- 小T should never cover operational data or push key actions below the fold.

### 2. Orange Smart Orb / System Core

Use when the page needs intelligent, system-level, or trustworthy data-processing atmosphere.

Allowed pages:

- Data analysis.
- Settings overview.
- App management.
- System health.
- AI insight cards.
- Account/points usage prediction.

Rules:

- Orange leads the visual identity.
- Blue may support as a secondary glow or system atmosphere.
- The orb/core should feel precise and controlled, not sci-fi or game-like.
- It works better than a mascot for serious data and system settings pages.

### 3. Module-Specific Micro Illustrations

Use when a module needs a more concrete business cue.

Allowed examples:

- POS/device setup.
- Payment configuration.
- Booking guidance.
- Staff permission.
- App activation.

Rules:

- Illustrations should follow the same soft, clean, orange-led style.
- No embedded text, fake UI copy, watermark, or accidental logo.
- Do not generate one-off styles per page.

## Page Mapping

| Page Type | Preferred Anchor |
| --- | --- |
| Cover / role selection | Flexible 小T + orange brand atmosphere |
| Owner home | 小T assistant or small assistant avatar + AI insight card |
| Staff home | Small assistant cue + app/module icons |
| Data analysis | Orange smart orb/system core + chart cards |
| Settings overview | System core + health/status chips |
| App management | Module illustrations + smart recommendation cards |
| Login/setup forms | Small helper cue only |
| POS ordering / checkout / transaction details | No large intelligent anchor; use restrained status/icons only |

## Anti-Patterns

- Do not force 小T into every page.
- Do not make 小T a fixed mascot if the product has not approved that role.
- Do not let the orange smart orb become a generic gradient blob.
- Do not use blue as the primary brand color.
- Do not mix unrelated mascot, 3D, line, and flat styles without a shared rule.
- Do not treat intelligent visuals as content replacement.
