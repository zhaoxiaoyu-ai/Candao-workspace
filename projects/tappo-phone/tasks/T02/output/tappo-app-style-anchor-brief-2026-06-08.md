## Extracted Style Position

Tappo Phone App should move toward a `light intelligent operations app` direction:

- Lighter than a POS backend.
- Cleaner than a traditional restaurant admin tool.
- Intelligent through subtle AI cues, not dark sci-fi decoration.
- Trustworthy through aligned data, restrained color, clear status, and predictable actions.
- Soft enough to feel friendly, but still data-first for daily restaurant operations.
- Brand color remains Tappo orange. The reference images guide UI tone, airiness, softness, card treatment, and intelligent visual cues; they do not change Tappo into a blue-first brand.

## Visual Language

Core words:

- Light
- Clean
- Intelligent
- Operational
- Trustworthy
- Soft
- Airy
- Data-first

Canvas:

- Use white, warm white, or very pale blue-gray as the default App canvas.
- Avoid heavy warm-cream dominance for App screens unless a specific page reference requires it.
- Use subtle blue or orange top wash only to create atmosphere; do not turn the app into a gradient poster.

Color:

- Primary brand and action color remains Tappo orange.
- Orange should lead CTAs, selected states, key account/balance moments, progress highlights, and brand recognition.
- Blue is a secondary `smart / AI / system / information` atmosphere color inspired by the references.
- Green is used for healthy, connected, successful, available, or automatic states.
- Red is only for operational risk, fault, shortage, or destructive states.
- Purple can appear only as a low-frequency AI/insight accent; it must not become the brand default.

Shape:

- Large cards: `20-24px` radius.
- Small module cards: `16-20px` radius.
- Icon containers: soft rounded square or circle, usually `40-48px`.
- Pills: full radius, used for state, status, and lightweight actions.

Depth:

- Use soft neutral shadows and pale glow.
- Avoid glassmorphism-heavy blur, dark cyber gradients, or noisy bokeh.
- Depth should make cards tactile, not decorative.

Typography:

- UI font stays `OPPO Sans` when available with Chinese/HK-safe fallback.
- Numeric emphasis uses `Montserrat` when available.
- Primary titles: `20-24px`.
- Section titles: `16px` semibold.
- Body: `14px`.
- Metadata: `12px`.
- Avoid `11px` except nav labels or compact status badges with proven legibility.

## UI Element Direction Before Page Rebuild

Do not continue by rebuilding pages one by one. Build and approve an App Style Kit first:

1. `Color and token board`: App canvas, primary orange, smart blue, success green, risk red, purple AI accent, text opacity ladder, shadow/elevation.
2. `Icon family`: line icons for nav and dense rows, soft filled module icons for dashboard cards, one consistent stroke/radius/gradient rule.
3. `AI/assistant visual`: a small mascot/orb/assistant token that can appear in home, insight, onboarding, and empty states without becoming a logo replacement.
4. `Operational cards`: sales/data card, AI insight card, asset/health card, exception card, transaction card, list row, status badge, progress bar.
5. `Illustration rules`: soft 3D/cloud/orb style allowed for AI and guidance areas; no embedded text, no fake UI copy, no watermark, no decorative overtake of data.
6. `Component specimens`: create neutral examples before applying them to real pages.

## Anti-Patterns

- Do not make every page orange.
- Do not let the reference images shift Tappo into a blue-first product.
- Do not default to Pad/POS warm cream for all App screens.
- Do not use randomly mixed icon sets.
- Do not generate one-off illustrations per page without a shared visual rule.
- Do not rebuild pages before tokens, icons, cards, and illustration direction are approved.
- Do not let mascot or AI visuals cover or weaken business content.
